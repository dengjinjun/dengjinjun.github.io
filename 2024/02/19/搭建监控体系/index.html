  2. 计划和分析：会议分析，制定初步计划与文档

工作需求

从0搭建以下的监控体系：

需要监控的细节：

  1. 基础设施/硬件：温度、风扇转速、raid状态、网络设备等
  2. 系统基础指标：CPU、内存、磁盘、网络、负载等
  3. 服务监控：不同服务如nginx/php/tomcat/db/nfs/rsync等
  4. 业务/应用监控：检查业务/应用是否可用，书写专用页面显示业务状态
  5. API接口/网关：curl/wget监控接口是否可用

其他监控：

  1. DNS/CDN监控：ping.chinaz.com(免费)、听云/应用宝(商业)
  2. pv/uv/ip监控：第三方监控代码，如百度统计等
  3. 用户行为监控：piwik/maomo等
  4. 舆情监控：第三方公司做

日志中状态码的数量监控

获取nginx日志中每种http状态码的数量，这个需要自定义监控项了

vim /server/scripts/log_status_code_count.sh

#!/bin/bash
# 脚本中传入状态码参数，awk取日志中返回此状态码的访问记录的总数

LOG=/var/log/nginx/access.log
awk -v code=$1 -v i=0 '$9==code{i++}END{print i}' $LOG

然后在zabbix-agent2的变量配置文件中写入

vim /etc/zabbix/zabbix_agent2.d/nginx.conf

UserParameter=nginx.status.code[*],sudo sh /server/scripts/log_status_code_count.sh "$1"

在服务端测试

zabbix_get -s lb401 01 -k nginx.status.code[200]

然后在web界面创建监控项即可。

同理，可以创建404、403等状态码的监控项，并设置触发器和图形。

TCP状态的数量监控

此处只写脚本内容

#!/bin/bash
# 利用ss工具，取80端口的监听状态、建立连接状态、等待关闭状态的数量
status=$1
port="80"

case "$status" in
    listen) ss -ant | awk -v p=":$port$" '$4~p' | grep -ic listen ;;
    estab) ss -ant | awk -v p=":$port$" '$4~p' | grep -ic estab ;;
    time-wait) ss -ant | awk -v p=":$port$" '$4~p' | grep -ic time-wait ;;
    *) echo "please input: listen|estab|time-wait"
easc

SSL证书过期时间监控

一般SSL证书到期之日前一个月告警

#!/bin/bash
# 获取证书内容，并且用awk工具取得截至日期，使用date命令将截至日期和今天的日期都转换为秒数进行计算，然后再转换为天数

Date=`openssl x509 -in /root/ca-bundle.trust.crt -noout -text | awk '/After/{for(i=4; i<=NF; i++) printf("%s%s", $i, (i<NF) ? " " : "\n")}'`
deadline_date=`date -d "$Date" +%s`
today=`date +%s`
remain_sec=`echo $deadline_date - $today | bc`
remain_day=`echo $remain_sec/60/60/24 | bc`
echo $remain_day

端口的监控

内置键值：net.tcp.port[<ip>,port]，检查是否可以将TCP连接到指定端口

ip：IP地址

port：端口号

进程数的监控

内置键值：proc.num[<name>,<user>,<state>,<cmdline>]获取进程数量

name：进程名称

user：用户名

state：进程状态，disk(不间断休眠)/run(运行中)/sleep(间断休眠)/trace(停止)/zomb(僵尸)

cmdline：按进程运行命令行过滤，这是一个正则表达式

    与nginx状态的监控类似，在nginx配置文件中先添加显示php状态的网页

    # 在nginx配置文件中增加此区段
location /php_status {
        fastcgi_pass 127.0.0.1:9000;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }

    在php的配置文件中定义此URL路径

    vim /etc/php-fpm.d/www.conf		# 找到`;pm.status_path`选项，修改为如下值，与nginx文件中定义的路径一致即可

pm.status_path = /php_status

web场景监控

许多时候我们需要监测一些web场景，比如监测网站后端的php是否连接上了数据库以及一些网站的API，经常用到web场景的监测

  1. 创建监控网页

    准备好一个文件mysqli.php，放到网站的访问目录中，文件内容如下

    <?php
    $link_id=mysqli_connect('db01','blog','123456');
    if($link_id){
        echo "mysql successful by blog"
    }else{
        echo mysqli_error();
    }
?>

  2. 创建web场景监控

    web界面侧边栏>配置>主机，选择一个主机，上方选项中选择web场景，右上角创建web场景，填写以下信息

    在步骤中，添加一个步骤，URL填写访问监控文件的URL

    填写要访问的主机头，要求的字符串和要求的状态码，点击添加即可

  3. 查看数据

    web页面侧边栏>监测>主机，点击相应主机的web监测，点击刚刚添加web监测场景，可以查看状态
