概述

关于共有镜像Docker Hub

Docker Hub是一个公共的Docker镜像仓库，使用前需要在Docker Hub官网注册一个账号，创建一个仓库（Repository）,比如test

需要注意的是，镜像名应该是用户名/仓库名:标签，这样才能正确地拉取和推送镜像

docker login		# 输入用户名和密码，登录Docker hub的账号，之后即可在Docker Hub上拉取和推送镜像

使用Docker私有仓库的原因

  1. 速度慢，官方的镜像仓库在国外，访问速度不言而喻;
  2. 安全性低，如果企业开发的核心代码上传到官方的镜像仓库，这意味着大家都能看到企业内部的代码;

主流私有仓库有哪些

  1. docker-registry：一个轻量级的镜像仓库，基本上不太占用内存，很适合学习环境中使用。
  2. harbor：宿主机最少需要2G内存，是企业的主流应用。

registry

部署启动

docker pull registry
docker run -d -p 5000:5000 --restart=always --name registry -v /data/docker_registry:/var/lib/registry registry

在浏览器访问http://服务端IP:5000/v2/_catalog即可查看仓库

上传镜像

# 首先要把私有镜像仓库的地址添加到受信任的仓库，才能使用https协议
vim /etc/docker/daemon.json		# 添加这句，受信任仓库的`域名:端口`或者`IP:端口`

"insecure-registries": ["registry:80"],

# 上传之前要先给镜像打上标签，镜像名中必须含有私有仓库地址信息才能正确上传
docker tag 原镜像名:标签 新镜像名:标签		# 比如 docker tag hello-world registry:80/hello:v1

# 推送
docker push 镜像名:标签		# 比如 docker push registry:80/hello:v1

在浏览器访问http://服务端IP:5000/v2/_catalog即可查看仓库中的镜像信息

其他镜像命令

# 其他镜像命令基本上没有太多差异，像拉取镜像、删除本地镜像，正常使用命令即可
docker pull 镜像名:标签		# 比如 docker pull registry:80/hello:v1

删除仓库中的镜像

# 进入容器内部，然后进入镜像的存储目录，可以发现有两个目录`blobs`和`repositories`
docker exec -it registry sh
cd /var/lib/registry/docker/registry/v2
ls

# 其中repositories存储的是镜像的元数据，blobs存储的是压缩后的镜像文件，需要分别删除
rm -rf repositories/镜像名/		# 删除镜像元数据，此时浏览器中已经查看不到该镜像的信息
registry garbage-collect /etc/docker/registry/config.yml 		# 回收无用文件，把没有元数据的镜像文件给清理掉

Harbor

harbor私有镜像仓库是在docker官方提供的”registry”基础之上二次开发，它包含了registry的核心功能。harbor相比docker的registry使用起来更加友好，基本上大多数操作都是在web界面中就能完成。

部署Harbor

注意：安装harbor之前，需要先安装docker-ce，docker-compose，container.io，并启动docker守护进程

# 下载
wget https://github.com/goharbor/harbor/releases/download/v2.3.0/harbor-offline-installer-v2.3.0.tgz

# 解压到指定目录
tar xf harbor-offline-installer-v2.3.0.tgz -C /softwares

# 编辑配置文件
cd /softwares/harbor		# 进入harbor软件目录
cp harbor.yml.tmpl harbor.yml		# 根据示例配置文件创建一个配置文件
vim harbor.yml		# 编辑配置文件

hostname: harbor.study		# 主机名，作为访问域名
http:
  port: 80		# http端口，默认80一般无需修改
harbor_admin_password: 123456		# 初始登录密码，用户名是admin
database:
  password: 123456		# 数据库密码
  max_idle_conns: 100
  max_open_conns: 900
data_volume: /data		# 存储卷的目录，会自动创建

# 安装harbor,等待安装完成即可
sh ./install.sh

在浏览器中访问harbor.study即可进入到harbor仓库的web页面

基本使用

# 同样的，需要把harbor仓库作为受信任的仓库地址添加到docker服务端的配置文件中，然后重启docker服务端
vim /etc/docker/daemon.json

  "insecure-registries": ["registry:80","study.harbor"],

systemctl restart docker

# 登录harbor私有仓库
docker login harbor.study

# 给镜像打标签，新镜像名中必须含有私有仓库的地址信息，以及项目名
docker tag 原镜像名:标签 新镜像名:标签
