配置菜单同步

需求：新疆梅花生物新搭建的环境，要同步吉林梅花生物的菜单

步骤：

  1. 找到吉林梅花生物的Inroad数据库，任务，生成脚本，选择对象，在表里面选择CommonAppMenuCustom 和CommonwebMenuCustom

  2. 设置脚本编写选项里面，高级，然后在最下面找到要编写脚本的数据的类型，选择仅限数据，然后点击确定，选择在新建查询窗口打开，然后把脚本复制下来

  3. 找到新疆梅花生物的Inroad数据库，直接新建查询，删除脚本中的第一行，添加两行，执行即可。

    truncate table CommonAppMenuCustom 
truncate table CommonwebMenuCustom

truncate table SafeOperationLicenseEvaluateConfigItem

调整作业票状态

需求：如图

步骤：

  1. 先找到东曹化工的Inroad数据库，然后执行一下查询语句，查询一下当前这张票的状态是什么，如下图所示，状态是3

    select status,* from SafeOperationLicenseRecord where workingbillno='TGC-2024-0530-0002'

  2. 作业票的所有的状态对应的数字如下所示

    -3 暂停  -2 挂起  -1 终止  0待开票  1 待评估  2 技术交底   3  审批中（流程中）  4 进行中   5完成待验收   6验收通过  7 待计划  8 待指派

  3. 修改到需要的状态，这个需求要调到交底阶段，所以应该是2

    update  SafeOperationLicenseRecord
set status='2'
where workingbillno='TGC-2024-0530-0002'

  4. 如果需要详细了解这张表的信息，可以执行

    exec Help_TableColumn  SafeOperationLicenseRecord

Inroad报错日志查询

  1. 看一下Inroad的配置文件中，把日志写到什么地方了，inroad一般应该是写到sqlserver里面了

  2. 连上sqlserver数据库，执行查询语句

    # 方式1
select * from LogDotNetApi where api != 'SafeInspectionPlanSend' order by c_createtime desc
# 方式2
select top 100 * from InroadLogInfo where [Level] != 2 order by CreateTime desc

数据库绑定DV3和DV5

需求：如图：

步骤：

  1. 找到客户的数据库，执行sql语句

    -- 这个数据库的名称要修改成客户的数据库名称
USE [Inroad-DEMO]
GO
ALTER ROLE [db_datareader] ADD MEMBER [InroadReadUser]
GO

GRANT EXECUTE ON [dbo].[CP_RiskControl_FourColorImageShow] TO [InroadReadUser]

GRANT EXECUTE ON [dbo].[CF_GetPinYin] TO [InroadReadUser]

    注意，如果发现报错，没有什么用户什么数据库主体什么的，可以查看一下数据库下面对应的安全性，用户，有没有一个[InroadReadUser]的用户，如果没有，还需要执行下面的sql语句

    -- 增加只读用户
USE [master]
GO
CREATE LOGIN [InroadReadUser] WITH PASSWORD=N'Inroad18121225109', DEFAULT_DATABASE=[master], CHECK_EXPIRATION=OFF, CHECK_POLICY=ON
GO
-- 这个数据库的名称要修改成客户的数据库名称
USE [Inroad-DEMO]
GO
CREATE USER [InroadReadUser] FOR LOGIN [InroadReadUser]
GO

  2. 登录DV3或者DV5的网站，去添加数据源
