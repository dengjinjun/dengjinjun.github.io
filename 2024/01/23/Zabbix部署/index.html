Zabbix部署

可查看官方文档：zabbix安装手册

zabbix部署环境：nginx 1.24.0、php 7.2、mysql 5.5

nginx

# 提前准备好nginx源
yum install -y nginx --enablerepo=[nginx-stable]

# 编辑站点文件
vim /etc/nginx/conf.d/zabbix.conf

# 输入以下内容
server {
    listen 80;
    server_name vm.zabbix;
    root /code/zabbix;
    location / {
        index index.php;
    }
    location ~ \.php$ {
        fastcgi_pass   127.0.0.1:9000;
        fastcgi_index  index.php;
        fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
        include        fastcgi_params;
    }
}

# 创建站点目录
mkdir -p /code/zabbix
chown -R nginx.nginx /code/zabbix

php

# 提前安装好php以及相关插件，更改用户和组
cd /etc/php-fpm.d/
egrep '^user|^group' www.conf				# 查看用户名和组名
sed -i '/^user/c user = nginx' www.conf		# 更改用户名
sed -i '/^group/c group = nginx' www.conf 	# 更改组名
egrep '^user|^group' www.conf				# 查看用户名和组名

# 创建session目录，默认应该是/var/lib/php/session
grep 'session.save_path' /etc/php-fpm.d/www.conf
mkdir -p /var/lib/php/session
chown -R nginx.nginx /var/lib/php/session

# 创建测试文件
vim /code/zabbix/index.php

# 输入以下内容
<?php
phpinfo();
?>

mariadb

# 安装数据库
yum install mariadc-server;

# 登录数据库，创建数据库并添加用户
create database zabbix character set utf8 collate utf8_bin;
grant all on zabbix.* to zabbix@'localhost' identified by '123456';

zabbix服务端

# 安装zabbix的yum源
rpm -Uvh https://mirror.tuna.tsinghua.edu.cn/zabbix/zabbix/5.0/rhel/7/x86_64/zabbix-release-5.0-1.el7.noarch.rpm

# 安装服务端和客户端
yum install -y zabbix-server zabbix-agent2

# 在数据库中导入数据
rpm -ql zabbix-server-mysql		# 查询rpm包中的文件，找到create.sql.gz文件，里面是建表的sql语句
zcat /usr/share/doc/zabbix-server-mysql-5.0.40/create.sql.gz | mysql -uzabbix -p123456		# 执行

# 修改zabbix-server配置文件，连接数据库
vim /etc/zabbix/zabbix_server.conf		# 编辑文件的如下配置
DBHost=localhost
DBName=zabbix
DBUser=zabbix
DBPassword=123456
gerp ^DB /etc/zabbix/zabbix_server.conf		# 查看确认

# 开启zabbix服务端
systemctl start zabbix-server
systemctl enable zabbix-server
ss -tunlp | grep zabbix		# 查看端口，服务端的端口是10051，客户端端口是10050

安装web插件

两种方式，可以根据官方手册通过yum安装；也可以使用源码包的方式安装。此处选择源码方式。

# 去官网下载源码包zabbix-5.0.40.tar.gz，解压
tar -xf zabbix-5.0.40.tar.gz

# 将ui目录中的文件，全部移动到nginx站点目录下
cd zabbix-5.0.40
mv ui/* /code/zabbix/

# 访问站点目录，完成可视化安装。（可能需要修改php.ini文件以及安装扩展）

zabbix客户端

以往zabbix的客户端是C语言开发的zabbix agent，以独立进程方式运行，后来在C语言的基础上增加了Go语言的部分，并且引入了多线程技术，能够承受更高的并发，减少CPU资源消耗，称为zabbix agent2。

# 按照zabbix客户端
yum install -y zabbix-agent2

# 编辑zabbix客户端配置文件，客户端需配置Server=服务端IP，服务端本机设置Server=127.0.0.1即可
vim /etc/zabbix/zabbix_agent2.conf

# 启动zabbix-agent2
systemctl enable zabbix-agent2
systemctl start zabbix-agent2

安装调试工具

为了确认zabbix服务端与客户端之间能够正常连通，一般使用调试工具zabbix-get

# 安装调试工具
yum install -y zabbix-get

# 使用，一般用到一个命令就行：`zabbix_get -s [客户端IP] -p [客户端端口] -k [状态信息键名]`
zabbix-get -s 192.168.110.137 -k system.hostname	# 此处获取客户端主机名，如果有正常返回说明成功

部署成功

Zabbix配置文件

官方手册传送门

服务端

|root@m01 ~|$ grep '^[a-Z]' /etc/zabbix/zabbix_server.conf 
LogFile=/var/log/zabbix/zabbix_server.log			# 日志文件
LogFileSize=0										# 日志文件大小，0代表不限制，不自动切割
PidFile=/var/run/zabbix/zabbix_server.pid			# 进程pid文件
SocketDir=/var/run/zabbix							# socket文件
DBHost=localhost									# 数据库主机
DBName=zabbix										# 数据库名
DBUser=zabbix										# 数据库用户名
DBPassword=123456									# 数据库用户密码
SNMPTrapperFile=/var/log/snmptrap/snmptrap.log		# SNMP相关日志
Timeout=4											# 外部检查的超时时间
AlertScriptsPath=/usr/lib/zabbix/alertscripts		# 告警脚本的路径
ExternalScripts=/usr/lib/zabbix/externalscripts		# 外部脚本存放路径，先不管
LogSlowQueries=3000									# 日志慢查询时间，超过3000ms会记入日志
StatsAllowedIP=127.0.0.1							# 谁可以查看服务端的状态

客户端

[root@web01 ~]$ grep '^[a-Z]' /etc/zabbix/zabbix_agent2.conf 
PidFile=/var/run/zabbix/zabbix_agent2.pid
LogFile=/var/log/zabbix/zabbix_agent2.log
LogFileSize=0
Server=192.168.110.128			# zabbix服务端IP 
ServerActive=127.0.0.1			# 主动模式下的服务端IP
Hostname=Zabbix server			# 客户端主机名
Include=/etc/zabbix/zabbix_agent2.d/*.conf		# 包含其他的配置文件，类似于nginx的include
ControlSocket=/tmp/agent.sock	# socket文件

GUI相关操作

初始账号Admin，初始密码zabbix，成功登录。

添加用户群组

  1. 进入zabbix监控的web界面，点击侧边栏的管理>用户群组，右上角创建用户群组，并填写相关信息

  2. 选择权限，可以给这个用户群组分配主机群组的权限，可稍后分配。完成后点击添加即可。

添加一个用户

  1. 进入zabbix监控的web界面，点击侧边栏的管理>用户，右上角创建用户，并填写相关信息

  2. 添加完成之后，即可使用此账号登录。

添加一个主机群组

  1. 进入zabbix监控的web界面，点击侧边栏的配置>主机群组，右上角创建主机群组。
  2. 填写群组名称，点击添加即可。

添加一台客户端主机

  1. 进入zabbix监控的web界面，点击侧边栏的配置>主机，右上角创建主机。

  2. 输入主机名称（需要与客户端Hostname相同），定义群组（一台主机可属于多个群组），输入客户端IP和端口，连接到IP。

  3. 点击菜单栏中的模板，在链接新模板中选择一个合适的模板，点击添加。

  4. 在主机界面的主机列表中，看到刚刚添加的主机的可用性变为绿色即可。回到监测界面的仪表盘，稍后刷新可看到主机数量增加。

解决图形界面的中文乱码问题

侧边栏，监测>最新数据，选择一个监控项的图形界面，可能会出现下面的乱码，这是由于zabbix的字体在Windows设备上无法正常显示导致的。

解决办法：

# 这是zabbix下的字体文件
|root@m01 ~|$ ll /code/zabbix/assets/fonts
-rw-r--r-- 1 nginx nginx 756072 11月 30 16:41 DejaVuSans.ttf

# 在Windows设备上找一个字体文件，替换上去就行了，比如Windows设备上的宋体 C:\Windows\Fonts\simsunb.ttf
|root@m01 ~|$ cp simsunb.ttf /code/zabbix/assets/fonts/DejaVuSans.ttf 

# 刷新web页面即可
