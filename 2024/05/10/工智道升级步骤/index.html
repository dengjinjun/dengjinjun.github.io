站点升级

  1. 在目标服务器上双击运行winagent，路径：C:\\tool\winagent.exe
  2. 在jenkins中搜索客户名称，找到客户名称对应的任务DEPLOY-客户名称-SITE，带参数构建，根据需要升级到的inroad的版本填写参数，当前仅支持inroad站点和OAA站点更新，后续可能会加上todo和lowcode
  3. 可选（如果需要升级积木的话）。在99lowcode机器上找到指定版本的lowcode的后端文件，路径在D:\InroadVesion\LowCode2.0Versions\版本号\发布文件\Backend，将其覆盖放到目标服务器的lowcode站点目录中。

数据库操作

inroad数据库升级

  1. 备份客户的inroad数据库和lowcode数据库

  2. 用redgate compare工具同步客户inroad数据库的结构，用redgate data compose工具同步客户inroad数据库的数据内容。

  3. 根据客户的当前版本，选择执行要执行的sql脚本，一般需要执行2023全年操作的sql，然后24年每个版本依次执行版本号_dataset.sql，脚本文件已放在了指定工具文件夹内。

todo数据库

建立一个todo的数据库，数据库名为InroadTodoService_客户名称，然后执行初始化脚本，脚本文件已放在了指定工具文件夹内。

lowcode数据库升级

按照版本升级的跨度，依次执行脚本，比如当前在2.3.1版本，要升级到2.3.4版本，那么就要依次执行2.3.2，2.3.3，2.3.4版本的sql脚本。脚本文件已放在了指定工具文件夹内。

配置文件操作

# Inroad的appsetting.json文件，Inroad键值内增加

 "TodoListAddress": "http://127.0.0.1:89/",		// 此后todo服务使用89端口
 "ComplexBoxHost": "http://103.229.213.171:90/",	//2024-0508	如果服务器端和络盒内网地址相通，配置内网地址
 "ComplexBoxHostRoutePrefix": "realTime",	//2024-0508     络盒配置无需改动

# lowcode的appsetting.json文件，根目录下增加todo相关配置

    "RemoteServices": {//远程服务
        "Todo": {
            "BaseUrl": "127.0.0.1:88" //待办服务的远程地址
        }
    }

# nginx增加转发路径

    location /todo/api {
        proxy_pass http://127.0.0.1:89/api/app;  # 此后todo服务使用89端口
    }
    location /realTime/api/ {
        proxy_pass http://103.229.212.125:1010/api;   # 使用络盒的后端地址，如果内网能通使用内网地址
    }

# oaa的appsetting.json文件，根目录增加域名配置
        
  "Domain": "https://demo.in-road.com"

升级&测试

重启所有站点，重启nginx，目标服务器内使用浏览器测试，确认以下项目正常：

  1. 访问域名正常
  2. 登陆注册正常
  3. 网站域名/vweb/#/orgnization/ToDoCenter 正常
  4. 网站域名/low-code-platform/home 正常
