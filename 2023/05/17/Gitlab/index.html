GitLab概述

GitLab 是一个用于仓库管理系统的开源项目，使用Git作为代码管理工具，并在此基础上搭建起来的web服务。

使用GitLab可以自己搭建一个类似于GitHub一样的仓库，但是GitLab有更完善的管理界面和权限控制。对于开源项目而言，GitHub 是代码托管的首选。但是从代码的私有性上来看，GitLab 是一个更好的选择。

访问GitLab有两个入口，一个是内置Nginx控制的web入口，另一个是SSH协议的GitLab Shell入口。

GitLab部署

安装GitLab依赖包

yum install -y curl wegt openssl-server postfix		# 安装依赖：访问站点的组件、ssl组件、邮件相关组件

# 在清华官方源中找到GitLab的rpm包安装

GitLab迁移与备份

手动备份

  • 更改配置

    # 编辑配置文件，通过`/Backup Settings`找到此行，修改备份相关配置

### Backup Settings
###! Docs: https://docs.gitlab.com/omnibus/settings/backups.html

gitlab_rails['manage_backup_path'] = true		# 开启自定义目录
gitlab_rails['backup_path'] = "/var/opt/gitlab/backups"		# 存储备份文件的目录
gitlab_rails['backup_archive_permissions'] = 0644		# 备份文件的权限
gitlab_rails['backup_keep_time'] = 604800		# 存储时间，以秒为单位

gitlab-ctl reconfigure		# 重启

  • 执行备份

    # 手动执行备份命令，会在备份目录下生成一个压缩包
gitlab-rake gitlab:backup:create		# 执行备份，此命令适用于12.1及以前版本
gitlab-backup create		# 执行备份，此命令适用于12.2及以后版本

# 注意：配置文件gitlab.rb以及密码文件gitlab-secret.json包含敏感信息，需要手动备份

自动备份

  • 更改配置

    # 同上

  • 编写脚本

    vim /server/scripts/backup-gitlab.sh

#!/bin/bash

# 执行备份
gitlab-rake gitlab:backup:create
# 备份配置文件以及密码文件
tar -zcf /backup/gitlab-conf-$(date +%F).tar.gz /etc/gitlab

  • 写入定时任务

    crontab -e

# 每天凌晨3点备份
00 03 * * * sh /server/scripts/backup-gitlab.sh &>/dev/null

恢复

# 先停止GitLab的写入类服务
gitlab-ctl stop puma
gitlab-ctl stop sidekip

# 执行恢复命令
gitlab-rake gitlab:backup:restore BACKUP=备份文件的压缩包名		# 执行恢复，此命令适用于12.1及以前版本
gitlab-backup restore BACKUP=备份文件的压缩包名		# 执行恢复，此命令适用于12.2及以后版本

# 重启
gitlab-ctl restart

# 注意：如果需要恢复配置文件和密码文件，可以找到单独备份的压缩包，手动替换这两个文件，然后执行`gitlab-ctl reconfigure`

GitLab迁移与升级

升级

GitLab升级的理由：

  1. 非常吸引人的新功能
  2. 安全漏洞和修复bug
  3. 标准化和自动化

GitLab升级的步骤：

GitLab的升级必须按照官方指定的升级顺序进行升级，不能够随意越级更新。注意要做好备份和快照。参考网页

迁移

GitLab跨服务器迁移有两种情况

  1. 同版本迁移，只需要备份好GitLab数据和配置及密码文件，然后在另一台机器上部署同版本的服务，恢复即可。
  2. 跨版本迁移，最好先在原来的机器上跨版本，然后再进行同版本迁移。

GitLab重置密码的方法

# 进入GitLab服务器，命令行操作
gitlab-rails console
user = User.where(id:1).first
user.password=12345678
user.password.confirmation=12345678
user.save
