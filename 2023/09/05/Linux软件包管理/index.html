软件包管理

与Windows系统类似，Linux系统上也可以安装各种应用程序，称为软件包。Linux中软件包分为三类：

分类 安装 特点
rpm包 预先编译打包，安装简单 通常软件的稳定版本会被官方做成rpm包，所以软件版本偏低
源码包 手动编译打包，安装繁琐 软件版本丰富，可选择性强
二进制包 解压即可使用 不能修改原代码

对于源码，如果是解释性语言（shell,python等）编写的源码包，需要：

  1. 下载解释器：bash，python解释器等
  2. 在自己的Linux主机上安装对应版本的解释器以及依赖包。
  3. 在解释器的基础上运行：python a.py

如果是编译型语言（c,go等）编写的源码包，需要：

1. 下载源码包
1. 在自己的Linux主机上安装好编译器
1. 用编译器把源码包编译成可执行的命令

RPM包管理

RPM包介绍

RPM 是Red-Hat Package Manager（RPM软件包管理器）的缩写，起源于RedHat，但其原始设计理念是开放式的，适用于 redhat / centos / redora / suse 。

RPM包的获取方式

  1. 本地的ISO镜像光盘
  2. 自行网络下载
  3. 联网从yum仓库中获取rpm包

通过RPM命令管理RPM包

  • 安装

    rpm -ivh [RPM包名全称]

  • 升级

    rpm -Uvh [RPM包名]

  • 卸载

    rpm -e [RPM包名]

  • 查询

    rpm -q [包名]				# 查询某个RPM包是否已安装
rpm -qa					# 查询系统中安装过的所有RPM包
rpm -qa | grep xxx		# 利用管道符筛选
rpm -qc [RPM包名]			# 查询指定RPM包的配置文件

    rpm -qi [RPM包名]			# 查询指定RPM包的描述信息
rpm -ql [RPM包名]			# 查询指定RPM包里所包含的文件
rpm -qf [文件路径]			# 查询系统中指定文件所属的软件包
rpm -qd [RPM包名]				# 查询指定RPM包的帮助文档

    在上述命令中加上 -p 选项，能够查看未安装RPM软件包的信息

通过yum命令管理RPM包

yum是一个存放了所有相关RPM软件包的文件夹，称为yum仓库。

  1. 文件下涵盖了所有相关的软件
  2. 文件夹下有一个文件记录了本文件夹所包含软件包的依赖关系

  • 查询

    yum repolist				# 查看可用仓库

    yum repolist all			# 查看所有仓库状态，包括启用和禁用状态的仓库

    yum list					# 查看可用仓库中的所有软件包

    yum grouplist				# 查看可用仓库中的所有软件包组

    yum provides [路径/命令名]			# 查看命令所属的软件包

  • 安装

    yum install [包名]				# 安装软件包
yum install [包名] -y				# 非交互式安装软件包

    yum groupinstall [软件组名]	-y		# 非交互式安装软件包组

  • 重装

    yum reinstall [包名]				# 重装软件包，通常在配置文件损坏时重装，谨慎操作

  • 更新

    yum check-update				# 检查软件包是否有更新

    yum update -y						# 更新所有已安装软件包，通常刚装完系统之后更新。
yum update [包名] -y					# 更新指定软件包

  • 缓存

    yum makecache					# 制作元数据缓存
yum clean all					# 清理元数据缓存

    yum的一个配置文件/etc/yum.conf，在文件中设置keepcache=1，即开启了软件包缓存，缓存目录为cachedir指定的目录。

  • 卸载

    yum remove [包名] -y				# 非交互式卸载软件包

    yum groupremove [软件组名] -y		# 非交互式卸载软件包组

  • 历史记录

    yum history					# 显示历史yum操作记录，会记录每条yum操作的id号、时间、具体操作、执行人和变更文件数

    yum history info [id]		# 查看某条操作的详细信息
yum history undo [id]		# 撤销执行某条操作

配置yum源

yum源有两种：

  1. 本地源：把本地的一个文件夹做成了yum源

  2. 网络源：

    1. base基础源
    2. epel扩展源
    3. 某一个软件包官网提供的源

开启yum软件包缓存

vim /etc/yum.conf				# 编辑yum配置文件添加以下内容


keepcahce=1						# 开启缓存
installonlypkgs=*				# 缓存所有依赖
cachedir=/var/cache/yum			# 设置缓存文件保存的目录

配置本地源

创建本地源方式1：

  1. 自己采集一系列软件包放在一个本地目录下

  2. 在此目录下创建仓库

    yum install createrepo			# 安装createrepo命令

创建本地源方式2：

  1. 挂载

    mount /dev/sr0 /opt				# 把一个含有rpm包的镜像文件或光驱挂载到一个目录下，此处镜像文件是/dev/sr0，挂载到了/opt目录下

  2. 进入本地目录/etc/yum.repos.d，再次目录下创建文件local.repo

    cd /etc/yum.repos.d
vim local.repo

  3. 添加内容

    [local]							# 本地yum源的名字
name="this is local repo"		# 说明信息
baseurl=file:///opt				# yum仓库所在目录，file://是固定格式，代表本地源，后面跟目录，此目录必定是repodata文件夹的父级目录
enabled=1						# 是否开启yum仓库
gpgcheck=0						# 是否开启校验

创建网络源

网上下载一个配置文件即可,比如：

wget -0 /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo

调整yum源优先级

yum install yum-plugin-priorities			# 安装yum优先级插件
cat /etc/yum/pluginconf.d/priorities		# 查看配置文件，确认插件是否开启
vim /etc/yum.repos.d/nginx.repo				# 编辑每个yum源repo文件,添加priorities的值
priorities=1								# priorities的值越小，优先级越高

源码包管理

  1. 下载源码并解压

    wget [下载地址]
tar xvf

  2. 预先安装编译器，安装依赖库

    yum -y install gcc gcc-c++ autoconf automake make      
yum -y install zlib zlib-devel openssl openssl-devel pcre pcre-devel
或者
LANG=C yum -y groupinstall "Development tools"

  3. 生成配置文件

    cd [源码包目录]
./configure --prefix=[自定义安装路径]			# 指定安装路径

    此过程中如果报错，需要执行make clean all,清理出错文件，然后再安装好提示的依赖库，然后重新执行生成配置文件的步骤

  4. 编译安装

    make && make install

  5. 清理安装包
