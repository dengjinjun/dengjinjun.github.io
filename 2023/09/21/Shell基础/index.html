Shell编程概念

Shell简介

Shell是一种命令行解释器，是在计算机操作系统中提供用户与操作系统交互的一种方式。通常Shell是指命令行界面，用户通过Shell界面可以输入命令并获得相应的操作结果。Shell负责解释和执行用户输入的命令，从而实现管理和控制进程、操作文件、进行系统管理等操作。

常见的Shell类型包括Bourne shell（/bin/sh）、Bash shell（/bin/bash）等。其中，Bash shell是Linux系统中应用最广泛的Shell类型，也是默认的Shell类型。

Shell脚本

一般情况下，Shell脚本以文本文件的形式存在。Shell脚本由若干条命令和控制结构组成，可以在命令行中直接执行，也可以在脚本中进行批量处理。

Shebang解释器声明

在Linux系统中，Shebang语法通常用于指定脚本的解释器类型，如bash、sh、python等等。Shebang在Linux系统中位于脚本文件的第一行，以”#”和”!”两个字符组成的特殊语法。例如，#!/bin/bash就是一条Shebang语法，指示操作系统用bash解释器来执行后续的脚本命令。通俗地说，Shebang就是一种告诉操作系统如何解释执行脚本文件的语法。当操作系统执行一个脚本文件时，会首先读取文件的第一行，如果第一行以Shebang语法开头，就会根据指定的解释器来执行脚本文件中的命令。

简短的讲，Shebang是指以”#!”开头的一行注释，它告诉系统使用哪个解释器来执行这个文件。在Linux中，常见的Shebang声明包括：

#!/bin/bash          # 使用Bash shell解释器
#!/usr/bin/python3   # 使用Python 3解释器

注释和文档

Shell脚本中使用#符号来表示注释，注释可以用于说明脚本的功能、参数含义等相关信息。另外，Shell脚本还可以通过”Here Document”结构来定义多行文档：

#!/bin/bash
<<EOF							# 注释开始
这是一个Shell脚本示例
作者：张三
日期：2023-04-26
EOF								# 注释结束

可执行权限与执行方式

为了让Shell脚本能够像普通命令那样执行，必须为其添加可执行权限。在终端中使用chmod命令来修改文件的权限：

chmod +x script.sh						# 添加执行权限
./script.sh								# 方式一。进入脚本所在目录，./ 表示在当前目录下寻找可执行文件
sh script.sh							# 方式二。作为解释器的参数，这种方式直接指定了解释器，不需要Shebang语法来指定。

Shell变量

变量定义和赋值

在Bash中，变量可以用来存储任意类型的数据。变量名由字母、数字和下划线组成，不能以数字开头。变量的定义和赋值可以在一行完成。

  • 变量赋值

    变量的赋值需要使用等号（=）操作符，等号两端不能有空格。如果变量的值包含空格或其他特殊字符，需要使用引号将其括起来。例如：

    greeting="Hello World"							# 变量定义+赋值

  • 只读变量

    使用 readonly 命令可以将变量定义为只读变量，只读变量的值不能被改变。

    name="dengpangpang"
readonly name

  • 删除变量

    使用 unset 命令可以删除变量，但是不能删除一个只读变量

    unset name

  • 变量类型

    运行shell时，会同时存在三种变量：

    1) 局部变量: 局部变量在脚本或命令中定义，仅在当前shell实例中有效，其他shell启动的程序不能访问局部变量。

    2) 环境变量: 环境变量是一种全局变量，可以在整个系统中被访问和修改。在Bash中，可以使用“export”命令将一个变量声明为环境变量。例如：
export PATH=/usr/local/bin:$PATH		# 在原来的PATH中，增加了一条环境变量 /usr/local/bin

    3) shell变量: shell变量是由shell程序设置的特殊变量。shell变量中有一部分是环境变量，有一部分是局部变量，这些变量保证了shell的正常运行

变量引用与拓展

  • 引用变量

    在Bash中，可以通过在变量名前加上“$”符号来引用变量。例如：

    greet="Hello World"
echo $greet								# 输出 Hello World
echo ${greet}							# 输出 Hello World，语法更加规范，花括号可以帮助解释器识别变量的边界,还能实现其他高级功能
echo ${greet/World/Universe}			# / 是替换符，将World换成了Universe，输出 Hello Universe

  • 单引号和双引号

    在Shell脚本中，使用最多的变量就是数字和字符串，字符串的使用往往伴随着 ‘单引号’ 和 “双引号” 。双引号里可以有变量,也可以出现转义字符，但是单引号就有一些限制。

    name='dengpangpnag'

单引号里的任何字符都会原样输出，单引号字符串中的变量是无效的.比如：
greet='hello, ${name} !'
echo $greet						# 输出结果：hello, ${name} !

单引号字串中不能出现单独一个的单引号（对单引号使用转义符后也不行），但可成对出现，作为字符串拼接使用。
greet='hello, '$name' !'
echo $greet						# 输出结果：hello, dengpangpang !

字符串

  • 获取字符串长度

    string="abcd"
echo $
